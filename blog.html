Discuss the scope of var, let, and const

In sort term, Scope mean where we can access the variable. Basically, scope are two types global scope and functional scope. When we can access the variable anywhere of the document then it is call global scope. And when we can not access the outside of the function where we declare the variable then it is called functional scope variable.

Var:
var is a global scope variable, because we can access the variable from anywhere, inside of the function and outside of the function. Also, var can be redeclare and reassign.

let:
let is a function scope variable, because we can not access the variable out side of a function but if the "let" variable declare outside of a function then it is converted to global scoped. Also, let variable value can be reassign but let variable can not redeclare.

const:
const is also a function scope variable. It's behavior almost same as the let variable, only major difference is const variable can not redeclare and reassign.

Tell us the use cases of null and undefined

Null:
When we set a variable value as empty then the variable returns "Null". When we assign "null" as a value to any variable, it means that it is empty or blank.
When we want any variable contain no value then we use "null", It is also use in various condition and operations.

Undefined:
When we declare a variable but do not set any value to this variable, then the variable returns "undefined". Also, when we write a function with taking parameter but returns nothing then it's returns "undefined" and try to access object property which dose not exist in the object. Mainly undefined and null are use in error handling.

What do you mean by REST API?

REST API:
REST means Representational State Transfer and API means Application Programming Interface. In sort term, I means REST API is when we sent request to the server base on "CRUD"(CREATE, READ, UPDATE, DELETE) operation to get data from the server, server mainly transfer a "JSON" file. This json file taking process is REST API.
